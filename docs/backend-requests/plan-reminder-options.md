# 计划提醒配置字典接口说明

## 背景

前端在迭代 #2 的提醒策略配置界面中，需要展示可选的触发时机、通知渠道以及收件人群组，并告知偏移量的校验范围。目前这些常量仍以 Mock 方式硬编码，难以随着后端能力扩展同步更新，因此需要提供统一的字典接口。

## 接口概述

| 操作 | HTTP 方法 | 路径 | 说明 |
| --- | --- | --- | --- |
| 获取提醒配置字典 | GET | `/api/v1/plans/reminder-options` | 返回触发时机、渠道、收件人选项以及偏移量范围 |

### 响应结构

```json
{
  "triggers": [
    { "id": "BEFORE_PLAN_START", "label": "计划开始前", "description": "按照计划开始时间提前触发提醒。" },
    { "id": "BEFORE_PLAN_END", "label": "计划结束前", "description": "按照计划结束时间提前触发提醒。" }
  ],
  "channels": [
    { "id": "EMAIL", "label": "邮件通知", "description": "向所选收件人发送模板化的邮件提醒。" },
    { "id": "IM", "label": "即时通信", "description": "通过配置的协作通道推送即时消息提醒。" },
    { "id": "SMS", "label": "短信", "description": "通过短信服务发送文本提醒。" }
  ],
  "recipientGroups": [
    { "id": "OWNER", "label": "计划负责人", "description": "提醒计划的主要负责人。" },
    { "id": "PARTICIPANTS", "label": "计划参与人", "description": "提醒所有被分配到节点的参与成员。" },
    { "id": "CUSTOM", "label": "自定义收件人", "description": "为本次提醒额外指定联系人。" }
  ],
  "minOffsetMinutes": 0,
  "maxOffsetMinutes": 1440,
  "defaultOffsetMinutes": 60
}
```

### 业务说明

- **触发时机**：当前支持以计划开始与结束为基准，后续若扩展节点层级或其它场景，只需在服务端追加枚举及多语言配置。
- **通知渠道**：邮件、即时通信、短信均对应现有的提醒模板能力；前端可据此渲染选择器与辅助说明。
- **收件人群组**：覆盖计划负责人、参与人以及自定义联系人。自定义联系人会在前端引导用户输入额外邮箱/账号。
- **偏移范围**：约束偏移量在 0～1440 分钟之间，默认推荐 60 分钟，便于前端设定校验与默认值。

## 验收要点

- 不同语言请求头下返回的 `label` 与 `description` 应匹配对应语言资源。
- 若后端新增提醒渠道或触发枚举，需要同步补充多语言文案与单元测试。
- 字典接口为静态数据，可使用 CDN/缓存策略减少重复请求，但需在后端更新后失效缓存。
